# AI Usage

BridgeKit is a demo system and portfolio artifact. AI was used to speed up production work, not to replace design, product, or accessibility judgment.

## Where AI helped

- Initial project scaffolding and file organization.
- First-pass component prop framing and story outlines.
- Drafting documentation structure so the artifact could stay coherent across code, Storybook, and markdown notes.

## Where human judgment stayed primary

- Deciding the right level of scope for a believable portfolio artifact.
- Choosing which component states were worth documenting and which ones would just add noise.
- Reviewing copy tone so the project stays plainspoken and credible.
- Checking keyboard behavior, status labeling, and table or drawer semantics.

## Working principle

AI can shorten the path from an idea to a working draft. It should not be treated as the source of truth for interaction quality, accessibility, or system design.

For this project, the useful pattern is:

1. Use AI to accelerate the first draft.
2. Review the result with design-system intent in mind.
3. Tighten the API, states, copy, and accessibility details by hand.

That is the workflow being demonstrated here.
